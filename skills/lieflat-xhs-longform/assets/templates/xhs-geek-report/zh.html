这版 Codex 最强的地方，是它会把任务拆进真实环境里做。它会查文件、跑命令、读输出，也会在看到首屏不够干净时继续改。

01 / 不是代码补全，是工作流代理

以前的 AI 编程体验，像你旁边有一个很快的代码片段生成器。新版 Codex 更像一个 agent：它能在本地项目里读上下文、管理步骤、调用工具，并把结果落成文件。

这让它适合做那些中间步骤很多的任务：重构、迁移、视觉稿、文档、自动化检查。你不用把每一步都拆给它，它会自己维持一条执行线。

02 / 我测了三件事

TEST A读现有风格，不破坏项目语法PASS
TEST B把内容做成可发布的小红书长文PASS
TEST C预览后按截图继续修排版PASS

我最喜欢的是第三点。很多工具停在“生成了”，Codex 更愿意走到“看过了、改过了、能交付”。这才是日常工作里最值钱的部分。

03 / 它的边界也很清楚

它不是事实机器。凡是涉及最新政策、价格、版本、论文或发布状态，都要让它回到官方来源。凡是涉及账号、发布、权限、删除，也应该由人做最后确认。

  • 给它明确目标，不要只给情绪。
  • 给它验收标准，例如“首屏不要露出残缺模块”。
  • 让它跑检查，而不是只交一段解释。
  • 让它先读项目，再开始写。
recommended_prompt = """ 先读现有代码和视觉风格。 做最小必要改动。 完成后本地预览并截图检查。 如果排版不完整，继续修到可交付。 """

04 / 结论

Codex 最新版的价值，不是替你多写几行代码，而是替你推进一整轮“从需求到文件”的工作。

如果你常做项目、内容页、自动化和文档，它会变成一个很顺手的工作台。它不完美，但已经足够像一个会收尾的工程同事。
