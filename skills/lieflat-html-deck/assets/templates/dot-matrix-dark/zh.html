Thesis

核心变化：模型开始承担“过程”。

过去的模型更像高质量函数调用：你拆任务、控步骤、校验结果。GPT-5.5 的差异在于它更早理解目标，更少请求引导，更稳定地规划、用工具、检查并继续。

结论：价值不只来自“单次答案质量”，而来自更少的人工调度、更少的往返、更少的中途失控。

Prompting Shift

旧提示词需要瘦身，而不是继续堆叠。

从“按步骤服从”转向“按结果负责”。

KEEP

保留产品契约、成功标准、允许的副作用、证据规则和最终输出形状。

DROP

删除过度流程化的微管理提示。除非路径本身很重要，否则让模型选择执行路线。

TUNE

从 medium reasoning 开始；只有评测证明有收益时才提升到 high 或 xhigh。

Product Implication

GPT-5.5 把“模型选择”变成运营问题。

Instant / low

轻量问答、分类、短轮语音、低延迟查询。重点是吞吐和成本。

Default / medium

多数生产工作流的起点：质量、可靠性、延迟、成本的平衡点。

Thinking / high

复杂规划、工具密集、长上下文推理；需要明确停止条件。

Pro / xhigh

高价值、低频、难验证任务。适合专家复核，而不是无脑全量替换。

建议：用 eval 做路由，而不是用“最强模型”做默认答案。
Economics

真正的成本项，是任务完成总成本。

单 token 价格不是最后账单；失败重试、人工接管、工具乱用、上下文漂移才是隐性成本。

Latency

OpenAI 称 GPT-5.5 在真实服务中匹配 GPT-5.4 的 per-token latency。

Tokens

同类 Codex 任务使用更少 token，工具密集型流程里节省会被放大。

Ops

少写“补丁式提示词”，多做任务评测、工具权限和失败恢复设计。

Safety / Control

能力越像执行者，治理就越要像系统工程。

PRE-DEPLOY

安全评估前置

系统卡强调完整预部署评估、Preparedness Framework，以及针对网络安全和生物能力的红队测试。

RUN-TIME

权限面要收窄

越会用工具，越需要清楚的工具描述、参数约束、证据规则、确认点和审计日志。

HUMAN

人类变成裁判

人不再逐步指挥，而是定义目标、边界和验收标准；高风险动作仍应由人确认。

Adoption Playbook

优先切入“多步、可验证、高摩擦”的工作。

Start here

代码修复、数据清理、资料研究、表格/文档生成、客服复杂工单。

Avoid first

低价值闲聊、极低延迟分类、无验收标准的开放式探索。

Measure

任务完成率、人工接管率、工具错误率、总 tokens、总耗时。

Guard

权限最小化、操作确认、证据引用、回滚路径、敏感任务分级。

一句话：不要先迁移所有 prompt；先选择一个“人类一直在补锅”的流程。
Strategic Read

GPT-5.5 的竞争点，是工作流完成度。

模型正在从“聪明文本层”下沉到“操作系统层”。

For apps

差异化会来自工具面、上下文记忆、评测数据和人机交接设计。

For teams

Prompt 工程变成 workflow engineering：谁定义目标，谁验证，谁授权。

For AI

下一阶段的体感进步，不是多会聊天，而是更少需要被管理。

Closing

不是更大的聊天框，而是更小的管理负担。

GPT-5.5 的产品问题不是“它能不能回答”，而是“它能不能在你少操心的情况下，把复杂工作推进到可验收状态”。
