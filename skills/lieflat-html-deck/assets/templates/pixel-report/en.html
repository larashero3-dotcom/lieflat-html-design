07 / Prompt syntax
rewrite modeoutcome contract

Legacy style

Follow these 37 steps exactly.
Think very carefully.
Never make a mistake.
Use tool A, then tool B, then ask me.
If anything is unclear, stop immediately.
Return a long explanation.

Old prompts often freeze the route and stop the model from using stronger planning.

GPT-5.5 style

Goal: produce a verifiable deliverable.
Success: tests pass and key claims have evidence.
Tools: read, edit, run, and browse as needed.
Constraints: do not touch unrelated files.
Escalate: destructive actions or missing critical data.
Output: concise summary plus verification.

The new prompt is a work contract: goal, evidence, permissions, risk boundary, and delivery format.

Stronger models need less ritual and clearer responsibility.
08 / Safety + governance
system cardenterprise control

The more capability looks like an operator, the more governance must look like systems engineering.

The system card emphasizes pre-deployment evaluation, the Preparedness Framework, and red-teaming; runtime still needs permissions, evidence rules, confirmations, and audit logs.

Permission scope

Use least privilege; confirm write, send, delete, purchase, and publish actions.

Audit trail

Keep inputs, tool calls, file changes, source evidence, and final versions.

Eval framework

Use real enterprise samples to measure success, hallucination, and error cost.

Human gate

Use human approval for legal, medical, financial, production, and high-impact decisions.

Enterprise deployment still needs a local control plane.
09 / Model routing
choice frameworkquality vs cost

Use Pro / xhigh

Maximum-precision tasks

Accept higher latency and cost for deeper reasoning and difficult engineering.

Use Instant / low

High-volume routine tasks

Support drafts, classification, format conversion, short summaries, and low-risk templates.

Use human review

Irreversible decisions

Legal commitments, money movement, medical advice, production deletion, and regulated disclosure.

Routing metric: expected value of better completion minus token, latency, and governance cost.
